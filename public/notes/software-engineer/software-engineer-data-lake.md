# Data Lake

> Central repository for raw structured and unstructured data at scale.

## What it is

- Cheap object storage (S3) + open formats (Parquet).
- Schema-on-read — flexible but governance harder.
- Fed by batch and [Change Data Capture](/learning/software-engineer-change-data-capture) streams.

## Why it matters

Understanding **Data Lake** helps you reason about trade-offs in production data systems — when to apply it, what breaks at scale, and how it connects to neighboring ideas in this section.

## In practice

- Map the concept to tools you use (PostgreSQL, Kafka, Redis, etc.).
- Look for metrics or symptoms that indicate misuse or missing optimization.
- Cross-read the section index: [MOC](/learning/devops-cloud).
