## Concepts

| Note | Summary |
| ---- | ------- |
| [Consensus](/learning/software-engineer-consensus) | Agreement among nodes on a single value or log order despite failures. |
| [Quorum](/learning/software-engineer-quorum) | Minimum number of nodes that must agree for a read or write to proceed. |
| [CAP Theorem](/learning/software-engineer-cap-theorem) | Under partition, a system must choose between consistency and availability. |
| [BASE](/learning/software-engineer-base) | Basically Available, Soft state, Eventually consistent — pragmatic alternative to ACID at scale. |
| [Split Brain](/learning/software-engineer-split-brain) | Two partitions both believe they are the leader — risk of divergent writes. |
