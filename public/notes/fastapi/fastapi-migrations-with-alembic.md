# Migrations with Alembic

**Alembic** is SQLAlchemy's migration tool — the FastAPI equivalent of Django's `makemigrations` / `migrate`. It versions your schema independently of FastAPI.

## Setup

```bash
pip install alembic
alembic init alembic
```

```text
project/
├── alembic/
│   ├── versions/       # Migration files
│   ├── env.py          # Migration runtime config
│   └── script.py.mako
├── alembic.ini
└── app/
    └── models/
```

## Configure `env.py` for Async

```python
# alembic/env.py (simplified)
from app.models import Base  # Import all models so metadata is complete
target_metadata = Base.metadata

def run_migrations_online():
    connectable = create_async_engine(DATABASE_URL)
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
```

Import **every** model in `env.py` or Alembic won't detect them.

## Workflow

```bash
# Generate migration from model changes
alembic revision --autogenerate -m "add users table"

# Review the generated file in alembic/versions/

# Apply migrations
alembic upgrade head

# Rollback one step
alembic downgrade -1

# Show history
alembic history
```

## Example Migration

```python
# alembic/versions/001_add_users.py
def upgrade() -> None:
    op.create_table(
        'users',
        sa.Column('id', sa.Integer(), primary_key=True),
        sa.Column('username', sa.String(50), nullable=False, unique=True),
        sa.Column('email', sa.String(255), nullable=False, unique=True),
        sa.Column('password_hash', sa.String(255), nullable=False),
    )

def downgrade() -> None:
    op.drop_table('users')
```

## Data Migrations

```python
from alembic import op
import sqlalchemy as sa

def upgrade():
    op.add_column('users', sa.Column('slug', sa.String(50), nullable=True))
    # Backfill in Python
    connection = op.get_bind()
    connection.execute(sa.text("UPDATE users SET slug = username WHERE slug IS NULL"))
    op.alter_column('users', 'slug', nullable=False)
```

## Django vs Alembic

| Django | Alembic |
|--------|---------|
| `makemigrations` | `alembic revision --autogenerate` |
| `migrate` | `alembic upgrade head` |
| `showmigrations` | `alembic history` |
| `sqlmigrate` | `alembic upgrade head --sql` |

## CI/CD

```bash
alembic upgrade head  # Run before or during deploy
```

Use `--sql` in review pipelines to inspect DDL without applying.

## Combat Tips

### ✅ DO
- Review autogenerated migrations — they're not always perfect
- Keep migrations in version control
- Test downgrade paths on staging

### ❌ DON'T
- Don't edit applied migrations in production teams
- Don't run migrations from inside request handlers
- Don't forget to import new models in `env.py`
