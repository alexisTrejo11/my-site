# 11. Scalability & Performance

> Growing systems — horizontal vs vertical scale, pooling, cache, backpressure.

← [Software Engineer MOC](/learning/software-engineer-master-moc)

---

## Concepts

| Note | Summary |
| ---- | ------- |
| [Horizontal Scaling](/learning/software-engineer-horizontal-scaling) | Add more machines/nodes to handle load — scale out. |
| [Vertical Scaling](/learning/software-engineer-vertical-scaling) | Bigger CPU/RAM/disk on one machine — scale up. |
| [Load Balancing](/learning/software-engineer-load-balancing) | Distribute requests across healthy backends. |
| [Connection Pooling](/learning/software-engineer-connection-pooling) | Reuse DB connections instead of opening per request. |
| [Caching](/learning/software-engineer-caching) | Store frequently accessed data in faster tier (memory, CDN). |
| [Backpressure](/learning/software-engineer-backpressure) | Slow producers when consumers are overloaded. |
