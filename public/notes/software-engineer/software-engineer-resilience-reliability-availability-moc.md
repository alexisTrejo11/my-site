## Concepts

| Note | Summary |
| ---- | ------- |
| [Failover](/learning/software-engineer-failover) | Automatic or manual switch to standby when primary fails. |
| [High Availability](/learning/software-engineer-high-availability) | HA — designing for continued service through component failure. |
| [Circuit Breaker](/learning/software-engineer-circuit-breaker) | Stop calling a failing dependency after threshold — fail fast. |
| [Throttling](/learning/software-engineer-throttling) | Gradually slow request rate to protect system capacity. |
| [Rate Limiting](/learning/software-engineer-rate-limiting) | Cap requests per client/IP/key in a time window. |
