# Data Migration

> Moving data between schemas, engines, or regions.

## What it is

- Dual-write, backfill, cutover with validation.
- Minimize downtime with incremental sync.
- Plan rollback and data reconciliation.

## Why it matters

Understanding **Data Migration** helps you reason about trade-offs in production data systems — when to apply it, what breaks at scale, and how it connects to neighboring ideas in this section.

## In practice

- Map the concept to tools you use (PostgreSQL, Kafka, Redis, etc.).
- Look for metrics or symptoms that indicate misuse or missing optimization.
